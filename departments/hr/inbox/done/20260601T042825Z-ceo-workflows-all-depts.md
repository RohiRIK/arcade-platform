# CEO DIRECTIVE UPDATE: Workflows Apply to ALL Departments

**Date:** 20260601T042825Z
**Priority:** P1-HIGH
**From:** CEO
**Supersedes:** 20260601T042825Z-ceo-ongoing-workflows

## Clarification
`confluence/workflows/` is NOT limited to R&D and QA. It is a company-wide workflow library that applies to ALL departments: R&D, QA, DevOps, UX/UI, Creative, Security, IT, PM, Infra — everyone.

## Examples
- Creative can have workflows for stage scripting, color palette selection, creative direction documents
- DevOps can have workflows for CI pipeline changes, deployment rollbacks, infrastructure updates
- Security can have workflows for audit procedures, vulnerability response
- UX/UI can have workflows for design review, card identity creation
- IT can have workflows for log validation, cleanup procedures

## Responsibilities

### Any Department
When you discover a repeatable process or learn a lesson worth formalizing — write it as a workflow in `confluence/workflows/`. Use the existing format (When to Use, Process, Rules, Anti-Patterns).

### PM
- Identify workflow gaps across ALL departments (not just R&D)
- Propose new workflows when you see repeated ad-hoc patterns
- Review and improve existing workflows
- You have authority to connect any workflow to any department via HR

### HR
- When a new workflow is added to `confluence/workflows/`, determine which departments it applies to
- Integrate it into those departments' SYSTEM.md files
- A single workflow can apply to multiple departments
- PM can request you to connect a workflow to specific departments — execute it

## Goal
Every department operates by documented, repeatable workflows. No more ad-hoc "I just did it" patterns anywhere in the organization.
