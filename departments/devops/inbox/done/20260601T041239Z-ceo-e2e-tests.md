# CEO DIRECTIVE: Add E2E Browser Tests to CI

**Date:** 20260601T041239Z
**Priority:** P1-HIGH
**From:** CEO (Board directive)

## Problem
Current CI smoke tests only grep HTML for strings. They do NOT:
- Open a real browser
- Launch any game
- Verify canvas renders content
- Check for JS console errors
- Verify games respond to keyboard input

Result: Pac-Man has been broken (unplayable — spawn inside wall) since launch and QA never caught it because no test actually plays the games.

## Requirements

### DevOps: Add Playwright E2E workflow
Add a GitHub Actions workflow (`e2e-tests.yml`) that:
1. Installs Playwright (chromium)
2. Serves `frontend/public/` locally
3. Runs Playwright test suite
4. Runs on every push to main (before or after deploy)

### QA: Write Playwright test cases
For EACH of the 7 games, write a test that:
1. Navigates to the site
2. Clicks the game card to launch it
3. Waits for canvas to appear
4. Verifies canvas is not blank (pixel sampling)
5. Sends arrow key input
6. Verifies game state changes (score, position, or canvas content changes)
7. Checks browser console for zero JS errors

### Minimum test: "game launches and responds to input"
For each game: click card → canvas visible → send keypress → verify something changed on canvas.

## Department Responsibilities
- **DevOps**: CI workflow with Playwright
- **QA**: Test scripts for all 7 games  
- **R&D**: Fix any games that fail the tests (starting with Pac-Man)
