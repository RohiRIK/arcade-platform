# QA → Infra: P1 Bug — nginx.conf Missing API Proxy

**Priority: P1**
**Date: 2026-05-29**

The frontend nginx.conf has no `location /api` proxy_pass rule. The frontend JS constructs a direct URL to port 3001, which:
1. Gets blocked by the CSP `default-src 'self'` header (different port = different origin)
2. Breaks LAN access (other devices can't reach the backend port)

**Required fix:** Add to `frontend/nginx.conf`:
```
location /api {
    proxy_pass http://backend:3001/api;
}
```
