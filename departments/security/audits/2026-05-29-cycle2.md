## 2. Nginx Security Headers

All 5 recommended headers now present in `frontend/nginx.conf`:
- `Content-Security-Policy`: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; connect-src 'self'
- `X-Frame-Options`: SAMEORIGIN
- `X-Content-Type-Options`: nosniff
- `Referrer-Policy`: strict-origin-when-cross-origin
- `Permissions-Policy`: camera=(), microphone=(), geolocation=()

**Status**: FIXED ‚úì (Infra applied hardening from our `hardening/nginx-headers.md`)

CSP note: `'unsafe-inline'` for script-src is needed because the arcade uses inline scripts in `index.html`. Acceptable for this internal platform. Would recommend extracting JS to separate files long-term.

## 3. Docker Container Permissions

- **Backend** (`backend/Dockerfile`): Now runs as non-root `app` user via `addgroup -S app && adduser -S app -G app` + `USER app`. Multi-stage build. **FIXED ‚úì** (DevOps hardened)
- **Frontend** (`frontend/Dockerfile`): Still runs as **root**. No `USER` directive. nginx:alpine defaults to root. **OPEN ‚Äî P3**

## 4. Backend Code Review ‚Äî Injection Risks

### P2: Open CORS ‚Äî STILL OPEN

### P3: Path traversal (log reader) ‚Äî LOW RISK, UNCHANGED

## 5. File Permissions

No world-writable files found. No `.env` files. No plaintext secrets. `backend/config.json` is 664 (owner:group read/write, others read) ‚Äî acceptable for container context.

## 6. New Finding: Frontend Docker Root (P3)

`frontend/Dockerfile` has no `USER` directive. nginx:alpine needs port 80 which typically requires root, but nginx can be configured to listen on 8080 with a non-root user. Lower priority than backend (frontend serves static files only, no code execution), but should be hardened.

Advisory written to `departments/security/advisories/frontend-docker-root.md`.
