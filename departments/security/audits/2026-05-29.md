# Security Audit — 2026-05-29 (Initial Scan)

## 1. Dependency Audit

**Status**: BLOCKED — no `package-lock.json` exists in `backend/`. `npm audit` requires a lockfile.  
**Dependencies**: express ^4.21.0, cors ^2.8.5 — both current, no known critical CVEs as of this date.  
**Action needed**: Generate lockfile (`npm i --package-lock-only` in backend/) — sent to DevOps.

## 2. Nginx CSP Headers

**Status**: MISSING — `frontend/nginx.conf` has zero security headers.  
Missing headers:
- `Content-Security-Policy` — no CSP at all, allows inline scripts, external resources, XSS vectors
- `X-Frame-Options` — clickjacking possible
- `X-Content-Type-Options` — MIME sniffing possible
- `Referrer-Policy` — leaks referrer data
- `Permissions-Policy` — no feature restrictions

**Severity**: Medium. This is an internal arcade platform, but CSP is baseline hygiene.  
**Action needed**: Hardening doc written to `departments/security/hardening/nginx-headers.md`, advisory sent to Infra.

## 3. Docker Container Permissions

**Backend Dockerfile** (`backend/Dockerfile`): Runs as **root**. No `USER` directive. The node:20-alpine image defaults to root.  
**Frontend Dockerfile** (`frontend/Dockerfile`): Runs as **root**. nginx:alpine defaults to root.  
**Severity**: Medium. Container escape + root = full host access.  
**Action needed**: Advisory sent to DevOps/Infra.

## 4. Backend Code Review — Injection Risks

### 4a. Arbitrary Config Overwrite (P2)
**File**: `backend/src/index.js`, lines 31-35  
`PUT /api/config` accepts any JSON body and merges it into app config via `Object.assign`, then writes to disk. No authentication, no input validation, no schema check.  
- An attacker can overwrite any config key
- Can write arbitrary JSON to `config.json` on disk
- `cors()` with no options = allow all origins, so any website can call this endpoint

### 4b. Path Traversal Risk (P3)
**File**: `backend/src/routes/logs.js`, line 6  
`LOGS_DIR` uses `path.join(process.cwd(), '..', 'logs')` — the log reader itself is safe (only reads `.json` files from a fixed directory via `readdirSync`), but no validation on filenames. Current code filters by `.endsWith('.json')` which mitigates traversal. Low risk.

### 4c. Open CORS (P2)
**File**: `backend/src/index.js`, line 13  
`app.use(cors())` — allows requests from any origin. Combined with the unauthenticated config write endpoint, any webpage can modify server config.

## 5. File Permissions

No `.env` files found. No plaintext secrets detected. `config.json` contains no credentials.
