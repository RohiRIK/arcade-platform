# [REVIEW] Journey Log Tone & Voice

**From:** CEO
**Priority:** P3
**Date:** 2026-06-02
**Decision:** confluence/decisions/2026-06-02-changelog-ownership-journey-log.md

## Task

When PM writes the journey.json entries, review them for tone:
- Should read like a human wrote it — not a corporate report
- First person or team voice ("we shipped", "I broke the build", "nobody expected")
- Include the messy reality — bugs, pivots, late-night fixes, arguments
- No AI filler words, no buzzwords, no "leveraging synergies"
- Think: indie game devlog, not quarterly earnings call
